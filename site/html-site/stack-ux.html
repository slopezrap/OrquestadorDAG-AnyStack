🧩 Stack-agnóstico de verdad

Stack y UX desacoplados del motor.

El orquestador no debe estar atado a un stack concreto. Cada app declara su stack en STACK_PROFILE.yaml y su experiencia de usuario en UX_CONTRACT.md. El mismo DAG puede construir React/Node, SwiftUI/Go, frontend/backend declarados, Vue/Django — sin reescribir hooks, locks, agentes ni journey gates.

STACK_PROFILE.yaml

Frameworks, paths, comandos dev/test, health URL, base de datos, workflow Git y enforcer de tokens. Es la fuente única que el motor consulta para decidir qué scanner aplicar y qué git-workflow usar.

UX_CONTRACT.md

Personas, inventario de pantallas, estados UI requeridos, rutas, datos reales esperados y verificación visual/productiva. Una slice de UI no se cierra hasta que loading, empty, error, permission y success están explícitos.

Plugins despachables

.claude/enforcers/ y .claude/git-workflows/ evitan hardcodear un framework concreto o un workflow Git concreto. scripts/git-workflow.sh es el dispatcher que lee el campo git_workflow de STACK_PROFILE.

Git workflow plugins

Dos workflows incluidos out-of-the-box. scripts/git-workflow.sh hace dispatch al apropiado según STACK_PROFILE.yaml → git_workflow.

push-to-main

Para repos donde el closer commitea directamente a main. Es el default histórico del orquestador. Plugin: .claude/git-workflows/push-to-main.sh.

pr-flow

Para repos con branch protection o trunk-based con review obligatorio. git push + gh pr create. Plugin: .claude/git-workflows/pr-flow.sh.

UX states obligatorios por slice de UI

El UX_CONTRACT.md exige que cada pantalla declare estos seis estados. Una slice de UI no es ✅ hasta que están implementados o explícitamente marcados N/A.

EstadoCuándo apareceDatos requeridos
loadingMientras la request está in-flightSkeleton o spinner con accessible label
emptyCuando el endpoint devuelve 0 resultadosCuenta sandbox sin datos persistidos
network errorHTTP 5xx o timeoutMock de red caída + retry CTA
validation errorHTTP 400 con field-level errorsInputs inválidos reales
permission deniedHTTP 401/403Cuenta sin rol requerido
successHTTP 2xx con datosCuenta sandbox poblada
Cambia el stack y solo cambian dos ficheros. STACK_PROFILE.yaml y los plugins de .claude/enforcers/ y .claude/git-workflows/. Los hooks, locks, agentes, journey gates y closer guardrail no se tocan. Eso es lo que hace al motor genuinamente stack-agnóstico, no decorativo.
