Componentes principales

Source of truth · 5 docs

Cinco markdown/yaml canónicos en docs/source-of-truth/: instrucciones.md, *_TECHNICAL_GUIDE.md, *_IMPLEMENTATION_CHECKLIST.md, STACK_PROFILE.yaml, UX_CONTRACT.md. El bootstrap deriva registry, work-items, phases y DAG. La matriz de adyacencia no se edita a mano.

Memoria runtime

CLAUDE_TASK_PACK, handoffs, evidence y reports por TASK_ID. La identidad de ejecución se toma sólo del entorno de ese terminal. Cada terminal lleva su propio scope mediante CLAUDE_ACTIVE_TASK_ID y task pack.

Hooks

Hooks Python: spawn_budget, write_scope_guard, docs_discrepancy, update_ledger, capture_subagent_stop, session_context.

Locks POSIX

fcntl.flock reentrante con depth counter. Orden global registry → runtime-state. Atomic write tmp+rename. Multiprocessing tests reales.

Agentes

14 agentes reales en .claude/agents/: sesión (main-orchestrator), bootstrap (document-analyzer, project-architect, task-planner), slice (planner, developer, validator, tester, debugger, slice-verifier, screen-journey-reviewer, closer), docs opcional (official-docs-researcher) y release (deployer).

Plugins stack-agnósticos

.claude/enforcers/ y .claude/git-workflows/ son plugins despachables desde STACK_PROFILE.yaml. design_tokens_v1 escanea según el framework declarado. push-to-main y pr-flow cubren los dos workflows Git habituales.

Vocabulario centralizado

Vocabulario centralizado en orchestrator-contract.json → trailer_schema.roles. Hook valida outcomes y next_status contra el schema; si el contrato falta, registra error y no muta lifecycle.

UX Contract

UX_CONTRACT.md declara personas, pantallas y estados UI requeridos. Una slice de UI no se cierra hasta que loading, empty, network error, validation error, permission denied y success están explícitos o marcados N/A.

El DAG solo agenda. No rebaja quality gates. Cada slice pasa por las mismas validaciones — lo único que cambia entre modo histórico y modo DAG es el scheduler y la posibilidad de usar varios terminales en paralelo.
