Framework hardening

El motor se prueba a sí mismo.

El producto sigue entrando por cinco source-of-truth. La capa nueva añade orchestrator-doctor, .claude/schemas/ y una golden app real para detectar drift del framework antes de construir una app.

Schemas internos

Validan artefactos generados: handoff de verify, runtime logs, task records, stack profile y domain-rule index. No son un sexto documento.

Doctor global

Comprueba agentes, comandos, modelos, hooks, source-of-truth, runtime reality contract y ausencia de límites obsoletos.

Golden contract AnyStack

La app de ejemplo usa Python/SQLite solo para CI sin dependencias. El contrato es portable a Flutter, React, Node, Python, Go o cualquier stack declarado en STACK_PROFILE.yaml.
