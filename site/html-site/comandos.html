Slash commands

Comandos principales.

Cada comando se invoca dentro de Claude Code. Los gates humanos (/verify-slice, /verify-journey) son intencionales: ningún commit a main pasa sin verificación real.

/next-wave Scheduler

Lista nodos ready seguros

Imprime exports copy/paste por terminal. Excluye nodos que pisan el mismo Conflict group o Write set. No hace claim, no muta estado.
/next-slice TASK_ID Pipeline

Claim atómico + pipeline

Ejecuta planner → developer + official-docs-researcher opcional → validator ‖ tester → debugger si falla y, si todo va verde, continúa automáticamente con verify-slice. No cierra Git: deja verified_pending_close para /closer manual. El planner inyecta STACK_PROFILE y Domain rule refs en el task pack.
/verify-slice TASK_ID Gate humano

Verificación con datos reales/proporcionados

Router mecánico + slice-verifier: hard reset, datos reales/proporcionados y reproducción humana con Chrome DevTools MCP, claude-in-chrome o Agent360 Browser MCP (`browser-mcp`). Si hay Docker Compose, antes del reset asigna puertos host libres por slice (CLAUDE_FRONTEND_PORT, CLAUDE_BACKEND_PORT, CLAUDE_DB_PORT, etc.); docker compose -p aísla nombres/redes/volúmenes, pero no puertos host. Si no hay MCP browser conectado, bloquea con browser_mcp_unavailable. Si queda verified, pasa por verified_pending_close; entonces tú ejecutas /closer.

/closer TASK_ID Manual

Cierre Git explícito por el usuario

Solo se ejecuta cuando /next-slice o /verify-slice dejaron la task en verified_pending_close. Hace report, baseline sync, commit, workflow Git configurado, limpieza runtime Docker/Rancher de la slice y cleanup de worktrees. Es el único comando que invoca el subagente closer.

/auto-verify-slice Restringido

Helper estrecho, no flujo normal de journey

Comandos deterministas y producir evidencia. Rechaza si la slice cierra journey, toca auth, pagos, PII o migraciones críticas.

/verify-journey JID Manual

Gate end-to-end por journey

Rescate manual para journeys pendientes que no se verificaron inline en /verify-slice. Multi-pantalla, estados marginales (empty, error_network, permission_denied, deep_link, back, reload).

/phase-gate Pxx Bloqueante

Cierre real de phase

Bloquea avance si faltan tasks done, journeys sin verificar, reports/evidence/handoffs ausentes o follow-ups bloqueantes sin promover.

/register-followup Promote/waive

Trabajo nuevo descubierto en QA

Convierte hallazgos mayores en task DAG real. Sub-comandos: propose, promote, waive, list. Severities blocker|critical|high bloquean nuevas waves.

/revise-slice Corrección

Reabrir slice canónica

Sin cambiar el DAG ni crear IDs temporales. Corrige, revalida, re-verify y closer correctivo. Útil cuando aparece un bug menor tras el cierre.

/slice-maintain Housekeeping

Limpieza y compactación entre slices

Sub-comandos: clean (worktrees done, ledger antiguo, orphan task-packs) y compact (PROGRESS.md preservando últimas N slices, decisions y risks). Dry-run obligatorio por defecto.
